PART 1 — Unified JSON Log Format
Every detection module — gaze, latency, speech — writes to a single shared log file on the candidate machine. This unified format is the contract between all modules, the risk engine, and the Gemini API layer. Nothing outside this schema is accepted.

1.1 Master Session Log Schema

1.2 Universal Event Object
Every signal from every module is appended as a single event object. The schema is identical regardless of source. Modules never write custom formats.



1.3 Periodic Snapshot Object
Every 30 seconds, a coordinator process aggregates open signals into a snapshot. This lets the risk engine operate on windows without scanning the full event list each time.


1.4 Risk Timeline Entry
After every snapshot, the risk engine writes one risk timeline entry. This array becomes the primary input to the Gemini analysis layer at session end.


1.5 Signal Registry
All valid signal names are declared in this registry. Any event with an unregistered signal name is rejected by the log writer.

PART 4 — Symbolic Rule-Based Risk Engine
The rule engine is the first layer of analysis. It runs on the backend every time a new snapshot arrives. It uses classical data structures for efficiency and produces a list of fired rules and a composite score — which then feeds the Gemini layer.

4.1 Data Structures Used

4.2 Rule Definitions
Rules are evaluated in priority order. Each rule returns True/False and a severity level. When a rule fires, it is appended to the rules_fired list in the risk timeline entry.

PART 5 — Gemini API Analysis Layer
The Gemini layer runs in two modes: (1) a mid-session alert if the rule engine hits CRITICAL, and (2) a full post-session analysis that receives the complete JSON log and risk timeline. Gemini applies pattern recognition that symbolic rules cannot — cross-question trend analysis, temporal clustering, and natural language reasoning about speech content.

PART 6 — Snapshot Coordinator & Risk Loop
The SnapshotCoordinator runs on the backend. Every 30 seconds it reads the latest events from the session log, builds a snapshot, runs the rule engine, optionally calls Gemini, and writes results back to the log.
